# Conductor v1 — Roadmap

What shipped in the v1.0.0 release, one line per milestone.

| Milestone | Delivered |
|---|---|
| Skeleton | App scaffold, Redis client wrapper, `conductor.enqueue` API, `bench conductor worker`, `bench conductor doctor`, base DocTypes (`Conductor Queue`, `Conductor Job`, `Conductor Worker`), default queues seeded, `frappe.enqueue` opt-in shim. |
| Reliability core | Per-attempt audit (`Conductor Job Run`), declarative `RetryPolicy`, exponential/linear/fixed backoff with jitter, dispatch idempotency, execution lock, stalled-message reclaim via `XAUTOCLAIM`, dead-letter queue (`Conductor DLQ Entry`). |
| Scheduling | `bench conductor scheduler` singleton with leader election, delay drainer, cron loop, dead-worker reaper, `Conductor Schedule` DocType, `bench conductor schedule` CLI (list / enable / disable / run-now). |
| Dashboard | Standalone Vue 3 + Vite dashboard at `/conductor-dashboard` with overview, live job feed, job detail/timeline, DLQ browser with bulk retry, schedules view, workers panel, real-time updates over `publish_realtime`. |
| Workflows | DAG runner with `@conductor.workflow` decorator and `Step.depends_on`, atomic fan-in via Lua, reverse-topological compensations on terminal failure, definition versioning, workflow visualization (Mermaid) on the dashboard. |
| Multi-tenant polish | Pool-mode worker (`--sites=auto|A,B,C`) with per-job site context switching, per-tenant rate limits and concurrency caps via Redis token buckets, `bench conductor migrate-from-rq` one-shot importer, DLQ bulk-retry and per-tenant queue-depth CLI. |
| Dashboard UI refresh | Tailwind v4 migration of the Vue 3 dashboard, design-token system, accessibility pass, consistent component library. |
| User documentation | Diátaxis-shaped docs under `docs/` — getting-started tutorial, how-to guides (enqueue, schedule, triage, multi-tenant, workflows, RQ migration), reference (CLI, configuration, Python API), explanations (architecture, reliability, why Conductor). |
| KPI benchmarks | Reproducible Conductor-vs-RQ comparative harness covering throughput, duplicate-execution rate, audit completeness, transient-failure recovery. |
